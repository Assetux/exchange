# Assetux × 0x Protocol — Integration Announcement

**Assetux Exchange now sources swap quotes from both LI.FI and 0x Protocol simultaneously — and executes through whichever gives you more tokens.**

---

## Why we integrated 0x

No single DEX aggregator has the best rate on every swap. LI.FI is exceptional at cross-chain routing; 0x Protocol dominates single-chain EVM liquidity with its professional market-maker network and Matcha-powered routing engine. By running both in parallel and comparing outputs, Assetux can guarantee you always get the best available rate at the time of your quote.

---

## How it works

When you hit **GET QUOTE** on a same-chain EVM swap (e.g. ETH → USDC on Ethereum, or USDC → BNB on BNB Chain), the exchange fires two quote requests simultaneously:

1. **LI.FI** — aggregates Uniswap, Curve, Balancer, SushiSwap, and 30+ other DEXs
2. **0x Protocol** — taps professional market makers and its own aggregation layer

Both return the amount of output token you'd receive. The one that gives you more is shown as the winner (marked ★) and used for execution. The losing quote is still displayed so you can see the comparison.

```
LI.FI    0.9941 USDC
0x       1.0012 USDC  ★  ← best rate, will be used
```

For **cross-chain swaps** (e.g. ETH on Ethereum → SOL on Solana) and for **Solana swaps**, LI.FI is used exclusively since 0x only covers single-chain EVM.

## Execution via 0x

0x uses its **AllowanceHolder** router pattern. When a 0x quote is selected:

1. If the sell token is ERC-20, the widget checks the current allowance against `allowanceTarget`
2. If approval is needed, it prompts an `approve` transaction first
3. The swap transaction is sent directly to the 0x router contract — no intermediate hops

This is gas-efficient: 0x's settlement layer is optimized for minimal calldata and aggregates fills across multiple liquidity sources in a single transaction.

## Technical implementation

The integration is entirely server-side proxied to protect the API key:

```
Client → GET /api/quote/zerox?chainId=1&sellToken=0x…&buyToken=0x…&sellAmount=…&taker=0x…
Server → GET https://api.0x.org/swap/allowance-holder/quote (with 0x-api-key header)
```

The proxy normalises the `gas` field (0x returns it as an object; we flatten to a string) and passes the result back to the widget where it's compared with the LI.FI quote by raw `buyAmount`.

On the client, native tokens are remapped: LI.FI uses `0x0000…0000`, 0x uses `0xeeee…eeee` — the adapter handles this transparently.
